# Amazon Sales Data Analysis (SQL)
## Project Overview
In this project, I performed a comprehensive analysis of Amazon sales data. I integrated two distinct datasets—Orders and Products—using SQL Joins to derive actionable business insights.
## Tech Stack Used
Database: MySQL Workbench
Language: SQL
Dataset Source: Kaggle (CSV Files)
## Database Schema & Methodology
Data Loading: Created relational tables for Orders and Products and successfully imported the CSV data.
Data Cleaning: Resolved data integrity issues by handling inconsistent date formats and utilizing VARCHAR data types to fix import errors.
Data Analysis: Established a relationship between the two tables by performing JOIN operations on the Order_Id column.
## Key SQL Queries & Functions
I utilized several critical SQL commands to process the data:
DDL & DML: CREATE TABLE and data import techniques.
Joins: INNER JOIN to merge the Orders and Products datasets.
Aggregations: GROUP BY and SUM() to calculate total revenue distributed by Region and Category.
Sorting: ORDER BY to identify the top-performing categories and regions.
## Business Insights
Regional Performance: Identified which Region generates the highest total revenue.
Category Trends: Pinpointed the Product Categories with the highest demand and sales volume.
